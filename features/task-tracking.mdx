Gantt Maker's interactive task tracking system lets you manage task status with simple clicks, providing real-time visual feedback on project progress.

## Overview

Every task in your Gantt chart is interactive, allowing you to update status, edit details, and track progress without complex forms or menus.

<Tip>
  Click directly on tasks to update their status. The visual feedback is instant, keeping everyone aligned on project progress.
</Tip>

## Task Status Flow

Tasks move through a clear, three-stage lifecycle:

<Steps>
  <Step title="Pending">
    **Pending** tasks are planned but not yet started. They appear in the default state, ready for team members to begin work.
    
    Visual indicators:
    - Lighter color shading
    - No progress indicators
    - Full task bar visible on timeline
  </Step>
  
  <Step title="Work In Progress (WIP)">
    **WIP** tasks are actively being worked on. Click a pending task once to mark it as in progress.
    
    Visual indicators:
    - Distinct WIP marking or pattern
    - Partial progress visualization
    - Enhanced visibility in team member's color
  </Step>
  
  <Step title="Completed">
    **Completed** tasks are finished and verified. Click a WIP task again to mark it complete.
    
    Visual indicators:
    - Checkmark or completion badge
    - Muted or strikethrough styling
    - Full progress bar filled
  </Step>
</Steps>

<Note>
  The status flow is designed to match agile workflows: Pending → WIP → Completed. This simple progression keeps projects moving forward without status confusion.
</Note>

## Click-to-Update Interface

Update task status with simple, intuitive clicks:

### Basic Status Updates

<Steps>
  <Step title="Click a pending task">
    Click once on any pending task bar to mark it as Work In Progress (WIP).
  </Step>
  
  <Step title="Click again to complete">
    Click the same task (now in WIP state) to mark it as Completed.
  </Step>
  
  <Step title="See visual changes">
    Watch the task's appearance update instantly to reflect its new status.
  </Step>
</Steps>

### Advanced Status Management

For more control over task status:

- **Right-click or long-press** for status options menu
- **Status filters** to view only pending, WIP, or completed tasks
- **Bulk updates** to change status for multiple related tasks
- **Status history** to see when tasks transitioned between states

<Info>
  All status changes are saved automatically to your browser's local storage, ensuring no progress is lost even if you close the application.
</Info>

## Editing Task Details

Modify task information directly in the Gantt chart:

### Editable Task Properties

<Steps>
  <Step title="Select the task">
    Click on a task bar to select it for editing.
  </Step>
  
  <Step title="Open edit mode">
    Double-click or use the edit button to open task details.
  </Step>
  
  <Step title="Update information">
    Modify any task property:
    - Task name and description
    - Start and end dates/times
    - Assigned team member
    - Priority or category labels
  </Step>
  
  <Step title="Save changes">
    Changes save automatically and update the visual representation immediately.
  </Step>
</Steps>

### Quick Edit Features

- **Drag to resize**: Adjust task duration by dragging edges
- **Drag to move**: Shift task timing by dragging the entire bar
- **Click to reassign**: Change team member assignment with a click
- **Inline editing**: Edit task names directly in the chart view

<Tip>
  Use drag-and-drop editing to quickly adjust schedules during planning sessions. Changes are reflected instantly across all views.
</Tip>

## Best Practices

### Keeping Status Current

- **Daily updates**: Encourage team to update task status at least once per day
- **Stand-up reviews**: Use visual Gantt chart during daily stand-ups
- **End-of-day ritual**: Mark completed tasks before logging off
- **Mobile access**: Update status from mobile devices when working remotely

### Effective Status Progression

- **Start when you start**: Mark tasks WIP immediately when beginning work
- **Complete when done**: Don't wait to mark tasks completed
- **Avoid WIP buildup**: Limit simultaneous WIP tasks per team member
- **Review pending queue**: Regularly check upcoming pending tasks

### Visual Progress Management

- Use completion percentage to gauge sprint/project health
- Compare completed tasks against timeline expectations
- Identify team members who may need support (many WIP, few completed)
- Celebrate progress by reviewing completed task sections
